# Color System

> Building trust through color rather than decoration.

---

# Purpose

Color is one of the strongest emotional signals in interface design.

Vivaha does not use color to attract attention.

Vivaha uses color to create calm, confidence and clarity.

Every color should have meaning.

If a color exists only because it looks beautiful,

it should be removed.

---

# Philosophy

Color should support the experience.

Never dominate it.

Users should remember

People

Stories

Conversations

Relationships

Not gradients.

Not buttons.

Not colors.

The interface disappears.

Human connection remains.

---

# Emotional Palette

Vivaha is built around five emotional qualities.

Warmth

↓

Trust

↓

Calm

↓

Nature

↓

Light

Every color should reinforce one or more of these qualities.

---

# Primary Color

The primary color communicates trust.

It appears on

Primary buttons

Links

Active navigation

Interactive controls

Important highlights

It should never become visually aggressive.

The primary color should feel warm rather than corporate.

---

# Secondary Color

The secondary palette supports content.

Examples

Information cards

Section dividers

Navigation

Filters

Icons

Secondary actions

It should quietly support the primary palette.

---

# Neutral Palette

Neutrals define most of Vivaha.

Approximately

80–90% of the interface should consist of neutral tones.

Examples

Backgrounds

Typography

Cards

Containers

Forms

Whitespace

Luxury comes from restraint.

Not saturation.

---

# Accent Palette

Accent colors exist only to communicate meaning.

Examples

Success

Warning

Information

Error

Premium

Accents should never dominate layouts.

---

# Background Strategy

The background is never pure white.

Instead use soft warm neutrals.

Imagine

Handmade paper.

Ivory walls.

Natural linen.

Morning sunlight.

These references create emotional warmth.

---

# Surface Strategy

Every surface belongs to one layer.

Canvas

↓

Section

↓

Card

↓

Interactive Surface

↓

Overlay

Every layer becomes slightly more elevated.

Never dramatically different.

---

# Typography Colors

Typography should remain highly readable.

Headings

Highest contrast.

Body

Comfortable contrast.

Metadata

Lower emphasis.

Disabled

Clearly inactive.

Never reduce readability for aesthetics.

---

# Premium Color

Premium experiences should feel crafted.

Not expensive.

Use

Soft champagne.

Muted metallic tones.

Warm highlights.

Avoid

Bright gold.

Yellow gradients.

Glossy effects.

Luxury is communicated through restraint.

---

# Success

Success communicates reassurance.

Examples

Profile Saved.

Verification Complete.

Interest Sent.

Use muted greens.

Never neon.

---

# Warning

Warnings communicate attention.

Not fear.

Use warm amber.

Avoid aggressive orange.

---

# Error

Errors communicate guidance.

Not punishment.

Muted red.

Clear typography.

Helpful messaging.

Never flashing red.

---

# Information

Information should feel calm.

Use soft blue-gray tones.

Never bright cyan.

---

# Color Hierarchy

Users should notice colors in this order.

Interactive Elements

↓

Important Status

↓

Navigation

↓

Supporting Details

↓

Decoration

Color should guide attention naturally.

---

# Color Distribution

Approximate visual balance.

Neutrals

85%

Primary

8%

Secondary

4%

Accent

3%

If accent colors dominate,

the interface becomes noisy.

---

# Contrast

Accessibility is mandatory.

All text must satisfy WCAG AA.

AAA wherever practical.

Decorative choices never reduce readability.

---

# Gradients

Gradients are used sparingly.

Allowed

Large hero backgrounds.

Soft lighting.

Premium surfaces.

Forbidden

Buttons.

Inputs.

Cards.

Navigation.

Everyday UI.

Gradients should feel atmospheric.

Never decorative.

---

# Shadows And Color

Shadows should not introduce noticeable color.

Prefer

Warm gray shadows.

Low opacity.

Natural diffusion.

Avoid colored shadows.

---

# Dark Mode

Dark mode is not part of Version 1.

If introduced,

it must be designed intentionally.

Never generated automatically.

---

# Seasonal Themes

Seasonal celebrations may introduce small color accents.

Examples

Diwali.

Holi.

Christmas.

Eid.

The core palette never changes.

Vivaha should always remain recognisable.

---

# AI Implementation Rules

When generating interfaces,

prefer neutral backgrounds.

Avoid highly saturated colors.

Use color only when it communicates meaning.

Never invent additional colors.

Never replace the defined palette.

Whitespace remains more important than color.

---

# Review Checklist

□ Does every color communicate purpose?

□ Is readability preserved?

□ Are accents used sparingly?

□ Does the interface feel warm?

□ Does it feel calm?

□ Is accessibility maintained?

□ Does the experience avoid visual noise?

□ Would these colors still feel timeless in ten years?

Only if every answer is "Yes"

should the palette be approved.
