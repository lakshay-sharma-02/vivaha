# Typography

> Typography is the voice of Vivaha.

---

# Purpose

Typography is not decoration.

Typography is communication.

It determines

Readability.

Trust.

Emotion.

Hierarchy.

Pace.

Understanding.

More than any other visual element,

typography defines whether an interface feels professional or amateur.

Vivaha treats typography as one of its most important design foundations.

---

# Philosophy

People arrive at Vivaha carrying uncertainty.

Typography should reduce that uncertainty.

Good typography feels

Calm.

Human.

Editorial.

Refined.

Trustworthy.

Never

Corporate.

Mechanical.

Aggressive.

Over-designed.

Typography should feel like reading a beautifully printed book.

Not a software dashboard.

---

# Design Goals

Typography should

Create hierarchy.

Encourage reading.

Reduce cognitive load.

Support accessibility.

Remain timeless.

Every decision should improve comprehension.

Not decoration.

---

# Typeface Philosophy

Vivaha uses two typefaces.

Display Typeface

For emotion.

Body Typeface

For clarity.

Each has one responsibility.

Never blur them.

---

# Display Typeface

Purpose

Headlines.

Editorial moments.

Hero sections.

Major quotes.

Section titles.

Characteristics

Elegant.

Timeless.

Warm.

Readable.

Human.

Never decorative.

Never ornate.

Never fashionable.

The display typeface should feel like premium editorial publishing.

---

# Body Typeface

Purpose

Paragraphs.

Forms.

Navigation.

Buttons.

Cards.

Tables.

Settings.

Messages.

Characteristics

Highly readable.

Neutral.

Friendly.

Modern.

Accessible.

The body typeface should disappear.

Users should focus on meaning,

not the font.

---

# Typography Hierarchy

Every page follows the same hierarchy.

```
Hero

↓

Page Title

↓

Section Title

↓

Card Title

↓

Body

↓

Supporting Text

↓

Metadata

↓

Caption
```

Users should instantly understand information hierarchy without relying on colour.

---

# Reading Comfort

Target

60–75 characters per line.

Comfortable line height.

Generous spacing.

Consistent rhythm.

Reading should never feel dense.

---

# Paragraph Width

Long paragraphs reduce comprehension.

Preferred maximum width

680px

Reading width should remain comfortable on every device.

---

# Headline Philosophy

Headlines should communicate one idea.

Examples

Good

"Meaningful relationships begin with trust."

Poor

"The Best Matrimonial Platform For Every Family Across India"

Short headlines create confidence.

---

# Body Copy

Body text should be

Simple.

Professional.

Respectful.

Conversational.

Avoid

Marketing language.

Buzzwords.

Technical terminology.

Overly emotional writing.

---

# Font Weights

Keep the system intentionally small.

Regular

Primary reading.

Medium

Labels.

Buttons.

Navigation.

Semibold

Titles.

Bold

Rare emphasis.

Avoid excessive font weight variation.

---

# Text Alignment

Body text

Left aligned.

Headlines

Left aligned by default.

Centered only when emotionally appropriate.

Never justify paragraphs.

---

# Letter Spacing

Use minimal tracking adjustments.

Headlines

Slight tightening.

Body

Default.

Buttons

Slight increase.

Avoid exaggerated spacing.

---

# Line Height

Reading comfort takes priority.

Body

≈1.6

Headlines

≈1.2

Captions

≈1.4

Whitespace between lines improves understanding.

---

# Numbers

Use tabular figures where comparison matters.

Examples

Pricing.

Statistics.

Profile information.

Age.

Height.

Consistency improves readability.

---

# Lists

Lists should remain simple.

Comfortable spacing.

Short sentences.

Clear indentation.

Never create visually crowded lists.

---

# Links

Links should feel trustworthy.

Avoid bright colours.

Avoid excessive underlines.

Hover should communicate availability,

not excitement.

---

# Buttons

Button labels should

Use Title Case.

Never ALL CAPS.

Never decorative fonts.

Typography should communicate confidence.

---

# Forms

Labels always remain visible.

Placeholders are examples,

not replacements.

Input typography should match body typography.

---

# Accessibility

Minimum body size

16px.

Comfortable contrast.

Support browser zoom.

Support dynamic text scaling.

Typography should remain readable for users of every age.

---

# Responsive Behaviour

Typography should scale smoothly.

Desktop

Editorial.

Tablet

Balanced.

Mobile

Comfortable.

Never dramatically shrink typography on mobile.

Readability always wins.

---

# Copy Tone

Every sentence should feel

Patient.

Helpful.

Respectful.

Warm.

Clear.

Avoid

Pressure.

Scarcity.

Sales language.

Fear.

Judgment.

---

# AI Implementation Rules

When generating interfaces

Use typography to create hierarchy.

Do not create hierarchy using colour alone.

Prefer fewer font sizes.

Prefer generous spacing.

Maintain consistent rhythm.

Never invent additional font families.

Never use decorative fonts.

---

# Review Checklist

□ Is the typography immediately readable?

□ Is the hierarchy obvious?

□ Are headlines concise?

□ Is body text comfortable?

□ Is accessibility maintained?

□ Does typography create trust?

□ Would this remain readable after thirty minutes?

□ Would parents feel comfortable reading it?

Only if every answer is "Yes"

should typography be approved.
