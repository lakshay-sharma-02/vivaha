# The 20 Commandments of Vivaha

> The shortest possible guide to building Vivaha correctly.

These commandments are immutable.

Every screen.

Every component.

Every interaction.

Every illustration.

Every animation.

Every line of code.

Every AI-generated interface.

Must follow them.

If a decision violates one commandment,

the decision is wrong.

---

# I

## Trust Above Everything

Trust is the product.

The interface exists only to communicate that trust.

Whenever beauty conflicts with trust,

trust wins.

---

# II

## Design Homes, Not Websites

Every page should feel like entering another room in a thoughtfully designed home.

Not another webpage.

Users should feel welcomed,

not impressed.

---

# III

## Hospitality Over Luxury

Luxury says

"Look at us."

Hospitality says

"Welcome."

Vivaha always chooses hospitality.

---

# IV

## Calm Is a Feature

Reduce noise.

Reduce stress.

Reduce urgency.

Reduce distraction.

Increase confidence.

Increase clarity.

Increase comfort.

---

# V

## People Are Always More Important Than UI

Never let

animations,

decorations,

gradients,

or technology

become more noticeable than people.

People are the product.

Everything else supports them.

---

# VI

## Authenticity Wins

Choose

real homes,

real smiles,

real stories,

real conversations,

real families.

Never create unrealistic perfection.

---

# VII

## Simplicity Is Earned

Remove before adding.

Clarify before decorating.

Simplify before expanding.

Complexity must always justify itself.

---

# VIII

## Whitespace Is Luxury

Do not fear empty space.

Whitespace creates elegance.

Whitespace creates confidence.

Whitespace allows people to think.

---

# IX

## Motion Explains

Animation must never exist for entertainment.

Motion should only

guide,

confirm,

teach,

or reassure.

Nothing else.

---

# X

## Typography Is More Important Than Decoration

Users read before they admire.

Excellent typography creates trust.

Decoration supports typography.

Never the reverse.

---

# XI

## Every Screen Solves One Problem

One screen.

One purpose.

One primary action.

One emotional objective.

Avoid competing priorities.

---

# XII

## Accessibility Is Design

Accessibility is not a later improvement.

Accessibility is part of the first design.

Every user deserves the same quality experience.

---

# XIII

## Performance Is Beauty

Fast feels premium.

Slow feels broken.

Every millisecond matters.

Every animation earns its cost.

---

# XIV

## AI Must Be Invisible

Users should never admire the AI.

They should simply feel

understood,

supported,

and guided.

---

# XV

## Consistency Creates Confidence

Buttons behave consistently.

Cards behave consistently.

Typography behaves consistently.

Spacing behaves consistently.

Consistency builds familiarity.

Familiarity builds trust.

---

# XVI

## Every Interaction Reduces Anxiety

Ask yourself

"What emotion does the user feel immediately after this interaction?"

If the answer is uncertainty,

improve it.

---

# XVII

## Timeless Over Trendy

Avoid trends.

Avoid hype.

Avoid visual fashion.

Design something that still feels correct ten years from now.

---

# XVIII

## Respect The User's Time

Never create unnecessary clicks.

Never create unnecessary waiting.

Never create unnecessary reading.

Every interaction should feel worthwhile.

---

# XIX

## Build Systems, Not Screens

Never solve today's problem only.

Create reusable solutions.

Strengthen the design system.

Reduce future work.

---

# XX

## Leave The Interface Better Than You Found It

Every contribution should improve

clarity,

consistency,

performance,

documentation,

or accessibility.

Never leave the system weaker.

---

# The Vivaha Test

Before approving any design,

ask these ten questions.

□ Does this increase trust?

□ Does it reduce anxiety?

□ Does it feel calm?

□ Is it authentic?

□ Is it accessible?

□ Is it simple?

□ Is it performant?

□ Is it timeless?

□ Does it respect the user?

□ Would I confidently show this to my own family?

If any answer is "No",

the work is not finished.

---

# AI Quick Rules

If you are an AI system generating Vivaha interfaces:

Always:

✓ Build calm interfaces.

✓ Prioritize trust.

✓ Use whitespace generously.

✓ Keep motion subtle.

✓ Use editorial typography.

✓ Design for people, not aesthetics.

✓ Reuse existing components.

✓ Respect accessibility.

Never:

✗ Add flashy animations.

✗ Add unnecessary decoration.

✗ Invent new visual languages.

✗ Chase trends.

✗ Prioritize beauty over usability.

✗ Manipulate users.

✗ Create visual clutter.

✗ Ignore the Design System.
