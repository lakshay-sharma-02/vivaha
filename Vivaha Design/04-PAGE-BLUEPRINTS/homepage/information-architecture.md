# Homepage Information Architecture

> Information Architecture determines not what users see, but when they see it. The order of information determines the quality of decisions.

---

# Purpose

The homepage exists to answer questions.

Not all at once.

One by one.

Information should unfold naturally.

Every section should answer the question created by the previous section.

This creates curiosity.

Curiosity creates attention.

Attention creates trust.

Trust creates action.

---

# Core Principle

Users should never ask

"What is this?"

after the first section.

Users should never ask

"Why should I care?"

after the second.

Users should never ask

"How does it work?"

after the third.

Users should never ask

"Can I trust it?"

after the fourth.

The homepage progressively removes uncertainty.

---

# Information Journey

Every visitor naturally asks the following questions.

```
What is Vivaha?

↓

Can I trust it?

↓

How does it work?

↓

Who is here?

↓

Does it actually help people?

↓

Will my family trust it?

↓

What does it cost?

↓

What if I still have questions?

↓

How do I begin?
```

Every homepage section exists to answer exactly one question.

↓

Success Stories

Membership CTA

---

# Cognitive Load

Every section should communicate

One idea.

Maximum.

Never mix

Trust

Pricing

Stories

Verification

inside the same section.

Clarity always wins.

---

# Scroll Behaviour

Scrolling should feel continuous.

Avoid

Abrupt background changes.

Harsh colour transitions.

Competing visual anchors.

Long empty gaps.

The experience should feel like walking through connected rooms.

---

# Transition Strategy

Every section transition should answer

"Why am I seeing this next?"

Transitions may use

Whitespace.

Editorial photography.

Natural gradients.

Botanical dividers.

Never decorative separators.

---

# Information Priority

The homepage should prioritise

People

↓

Trust

↓

Process

↓

Platform

↓

Pricing

Never reverse this order.

People are always more important than software.

---

# Desktop Layout

Large immersive sections.

Editorial spacing.

Wide imagery.

Comfortable reading width.

Visual breathing room.

Desktop should feel architectural.

---

# Tablet Layout

Preserve hierarchy.

Reduce spacing proportionally.

Never remove content.

Only reorganise.

---

# Mobile Layout

Single-column.

Comfortable scrolling.

Large touch targets.

Readable typography.

Natural spacing.

The emotional journey must remain identical.

---

# Performance Strategy

Load only what is immediately necessary.

Priority

Navigation

↓

Hero

↓

Trust

↓

Remaining sections

Lazy-load below-the-fold imagery.

Avoid layout shifts.

Maintain smooth scrolling.

---

# Accessibility Strategy

Every section must

Use semantic landmarks.

Support keyboard navigation.

Maintain heading hierarchy.

Support screen readers.

Respect reduced-motion preferences.

Accessibility follows the same information architecture.

# AI Implementation Rules

When generating the homepage

Always preserve

Section order.

Visual rhythm.

Emotional progression.

Information hierarchy.

Accessibility.

Performance.

Never

Invent additional sections.

Reorder existing sections.

Combine unrelated sections.

Skip trust-building content.

Optimise for aesthetics over understanding.

---

# Review Checklist

□ Does every section answer one question?

□ Does the emotional journey flow naturally?

□ Is information introduced progressively?

□ Is cognitive load controlled?

□ Does scrolling feel architectural?

□ Is accessibility preserved?

□ Is the page performant?

□ Would a first-time visitor understand Vivaha naturally?

Only if every answer is "Yes" should homepage implementation proceed.
