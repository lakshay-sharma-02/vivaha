# Homepage — Section 04

# Discover Preview

> Visitors should imagine themselves exploring Vivah before they ever create an account.

---

# Purpose

This section transforms understanding into imagination.

Users should think

"I can picture myself here."

Not

"I need to sign up to know what it looks like."

---

# Emotional Journey

```
Understanding

↓

Imagination

↓

Curiosity

↓

Interest
```

---

# Philosophy

Never hide the product.

Never overwhelm users.

Provide a carefully curated glimpse.

Enough to build curiosity.

Never enough to replace the actual experience.

---

# Section Structure

```
Headline

↓

Supporting Copy

↓

Featured Profiles

↓

Explore CTA
```

---

# Headline

Maximum

Eight words.

Examples

Meet Thoughtful Individuals

Discover Genuine Profiles

Meaningful Introductions Begin Here

---

# Supporting Copy

Maximum

Three lines.

Explain

Every profile emphasizes authenticity,

shared values,

and meaningful compatibility.

---

# Featured Profiles

Display

Six profiles.

Desktop

3 × 2 Grid.

Tablet

2 × 3 Grid.

Mobile

Horizontal carousel.

Never display endless scrolling.

---

# Profile Information

Show

Photo.

First Name.

Age.

City.

Profession.

Verification Badge.

One compatibility highlight.

Nothing else.

---

# Photography

Natural.

Daylight.

Authentic.

Indian families and individuals.

No studio photography.

No wedding portraits.

---

# Compatibility

Display only

One insight.

Examples

Shared Values

Common Languages

Family-Oriented

Similar Interests

Avoid percentages.

Avoid rankings.

---

# CTA

Examples

Explore Profiles

View More Profiles

Begin Discovering

---

# Visual Design

Large whitespace.

Editorial spacing.

Soft background.

Natural botanical accents.

No decorative gradients.

---

# Motion

Allowed

Card reveal.

Soft hover.

Small elevation.

Tiny image transition.

Forbidden

Card flipping.

Swipe animations.

Parallax.

Continuous movement.

---

# Accessibility

Every card

Keyboard accessible.

Alt text.

Readable.

High contrast.

---

# Performance

Images

Responsive.

Lazy loaded.

Optimized.

Skeleton placeholders.

No layout shifts.

---

# AI Implementation Rules

Always

Show six curated profiles.

Maintain editorial spacing.

Prioritize authenticity.

Never

Display fake engagement metrics.

Show popularity rankings.

Use swipe interactions.

Create dating-app aesthetics.

---

# Review Checklist

□ Do visitors understand what profiles look like?

□ Are profiles authentic?

□ Is information easy to scan?

□ Is compatibility explained naturally?

□ Is whitespace generous?

□ Does this encourage exploration?

Only if every answer is "Yes" should this section be approved.
