# Homepage Responsive Blueprint

> Responsiveness is not about shrinking layouts. It is about preserving the experience across every screen.

---

# Purpose

Every visitor deserves the same emotional journey regardless of device.

Desktop should feel immersive.

Tablet should feel balanced.

Mobile should feel effortless.

The experience changes.

The story does not.

---

# Philosophy

Never design Desktop first.

Never design Mobile first.

Design the experience first.

Every breakpoint simply adapts that experience.

# Accessibility

Reading order remains identical across breakpoints.

Semantic headings preserved.

Keyboard navigation unchanged.

Screen reader hierarchy unchanged.

---

# AI Implementation Rules

Always

Adapt layouts.

Preserve hierarchy.

Keep emotional flow.

Never

Hide important sections.

Remove trust elements.

Create mobile-only experiences.

Compromise accessibility.

---

# Review Checklist

□ Does every device tell the same story?

□ Is every section readable?

□ Are touch targets large enough?

□ Is scrolling natural?

□ Is performance maintained?

□ Is accessibility identical?

Only if every answer is "Yes" should the responsive implementation be approved.
