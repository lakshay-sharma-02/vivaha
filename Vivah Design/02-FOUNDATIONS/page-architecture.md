# Page Architecture

> Every page in Vivah is an environment, not a screen.

---

# Purpose

Users should never feel like they are navigating software.

They should feel like they are moving through thoughtfully designed spaces.

A page is not a collection of sections.

A page is an architectural journey.

Every page should have

Beginning.

Middle.

End.

Just like a conversation.

---

# Philosophy

Traditional websites stack content vertically.

Vivah constructs experiences spatially.

Users should feel they are walking through

Rooms.

Courtyards.

Hallways.

Gardens.

Libraries.

Living spaces.

Every transition should feel intentional.

---

# Architectural Thinking

Never ask

"What section comes next?"

Instead ask

"What space should the user enter next?"

This single question should influence every page.

# Structure

Footer.

# Content

Content communicates.

Typography.

Photography.

Illustration.

Profiles.

Stories.

Content should remain the focus.

Everything else supports it.

---

# Interaction

Interaction allows movement.

Buttons.

Forms.

Navigation.

Search.

Chat.

Interaction should never compete with content.

"Can I trust this platform?"

# The Rule of Rooms

Each section should feel like entering a new room.

A room always has

Purpose.

Mood.

Light.

Primary content.

Exit.

Users should always understand when they have entered a new part of the experience.

---

# Hallways

Transitions are hallways.

Hallways prepare users emotionally.

Use

Whitespace.

Soft gradients.

Photography.

Quotes.

Natural pauses.

Never abruptly switch contexts.

# Content Blocks

Each content block should communicate one idea.

Examples

Verification.

Stories.

Never mix unrelated ideas.

---

# Reading Width

Readable content should remain narrow.

Never allow long paragraphs to span the entire viewport.

Users read comfortably.

Not efficiently.

---

# Container Philosophy

Containers define rooms.

They should create order.

Never decoration.

Containers should feel invisible.

Users notice content.

Not boundaries.

---

# Alignment

Alignment communicates professionalism.

Elements should align to meaningful relationships.

Not arbitrary positions.

Good alignment disappears.

Poor alignment becomes obvious.

---

# Visual Anchors

Every section should contain one dominant visual anchor.

Examples

Portrait.

Illustration.

Editorial image.

Quote.

Never create competing anchors.

Attention should remain focused.

---

# Progressive Complexity

Pages should gradually become richer.

Beginning

Simple.

Middle

Informative.

End

Actionable.

Never begin with maximum complexity.

---

# Empty Space

Every page requires moments of silence.

Whitespace acts like breathing.

Do not fear empty areas.

They improve comprehension.

---

# Responsive Architecture

Desktop

Immersive.

Tablet

Focused.

Mobile

Essential.

Do not shrink desktop.

Redesign for smaller screens.

---

# Accessibility

Architecture must remain understandable without visuals.

Semantic headings.

Logical reading order.

Keyboard navigation.

Visible focus.

Screen reader support.

Every structural decision supports accessibility.

---

# Performance

Architecture should encourage performance.

Lazy loading.

Predictable layouts.

Minimal layout shifts.

Efficient rendering.

Beautiful pages should remain fast.

---

# AI Implementation Rules

When generating pages,

identify

The page purpose.

The emotional objective.

The primary action.

The architectural sequence.

Never generate disconnected sections.

Never introduce visual clutter.

Treat every page like a sequence of spaces.

Not stacked components.
