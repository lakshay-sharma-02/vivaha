# Grid System

> Grids create invisible order. Users should notice the content, never the grid.

---

# Purpose

The grid system establishes visual order across every page of Vivah.

It determines

Alignment.

Consistency.

Balance.

Reading rhythm.

Responsiveness.

A good grid is invisible.

A poor grid becomes immediately noticeable.

The objective is to create interfaces that feel naturally organised rather than mechanically structured.

---

# Philosophy

Vivah is inspired by architecture.

Architects begin with structure.

Decoration comes later.

The grid is our architectural foundation.

Everything sits on top of it.

Never break the grid simply for visual effect.

---

# Principle

Alignment creates trust.

Misalignment creates doubt.

Users may never consciously recognise alignment,

but they immediately recognise inconsistency.

Every component should align to a common structural rhythm.

---

# Maximum Content Width

Large monitors should never produce unreadably wide layouts.

Recommended maximum container

1440px

Reading content

680–760px

Forms

520–640px

Cards

320–420px

Profiles

Optimised around comfortable reading rather than maximum density.

---

# Column System

Desktop

12 Columns

Tablet

8 Columns

Mobile

4 Columns

Never invent custom grids for individual pages.

Consistency is more valuable than novelty.

---

# Gutters

Desktop

32px

Tablet

24px

Mobile

16px

Spacing should feel generous without wasting space.

---

# Margins

Desktop

80px

Tablet

48px

Mobile

20px

Margins create breathing room between architecture and the viewport.

---

# Baseline Rhythm

Vertical rhythm should follow the spacing system.

All sections should align to the baseline whenever practical.

Avoid arbitrary vertical offsets.

---

# Reading Columns

Long-form text should never span the full content width.

Constrain reading areas.

Users read paragraphs.

Not screens.

---

# Card Alignment

Cards should always align to the underlying grid.

Never manually offset cards for visual experimentation.

Predictability improves scanning.

---

# Hero Layout

The Hero occupies the full architectural width.

Content remains constrained.

Atmosphere extends.

Content does not.

---

# Image Alignment

Photography should align with content.

Avoid floating images that ignore structural rhythm.

Images reinforce the architecture.

---

# White Space

Whitespace should exist within the grid,

not outside of it.

Large empty regions should feel intentional.

Never accidental.

---

# Responsive Behaviour

The grid adapts.

It never collapses randomly.

Desktop

12

↓

Tablet

8

↓

Mobile

4

Spacing reduces proportionally.

Hierarchy remains unchanged.

---

# Accessibility

The grid should never prevent

Zoom.

Reflow.

Dynamic text sizing.

Keyboard navigation.

Accessibility takes priority over rigid alignment.

---

# Performance

The grid should be implemented using CSS Grid and Flexbox.

Avoid deeply nested layout structures.

Keep layout calculations simple.

Predictable layouts reduce rendering cost.

---

# Implementation Notes

Use design tokens for

Columns

Gutters

Margins

Container widths

Never hardcode layout values inside components.

Components inherit from the grid.

---

# AI Implementation Rules

Never invent new container widths.

Never create page-specific grids.

Always align to the design system.

Use the grid to organise content,

not decorate it.

When uncertain,

choose simpler alignment.

---

# Review Checklist

□ Does everything align?

□ Is reading comfortable?

□ Does the grid disappear behind the content?

□ Is responsiveness preserved?

□ Are margins consistent?

□ Is whitespace intentional?

□ Does the layout feel architectural?

Only if every answer is "Yes"

should the layout be approved.
